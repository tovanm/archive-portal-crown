# archive-portal-crown

一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态页面文件的集中存储与展示入口。

## 项目简介

本项目旨在提供一个简洁、可维护的 HTML 页面归档方案。你可以将独立的 HTML 文件（如信息页、工具页、存档页等）存放于此，并通过仓库的 Pages 功能或直接浏览文件进行访问。

适合需要长期保存、整理或分享的静态页面内容。

## 目录说明

```
archive-portal-crown/
├── index.html          # 主入口页面（可选）
├── pages/              # 存放归档的 HTML 页面文件
│   ├── example-1.html
│   ├── example-2.html
│   └── ...
├── assets/             # 页面引用的公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：所有独立 HTML 归档页面均放置在此目录下，建议按类别或用途命名。
- `assets/`：存放页面公共资源，如样式、脚本和图片文件。
- `index.html`：可选的主页，可设置为导航页或说明页。

## 页面归档说明

- 每个 HTML 文件应保持独立自包含，尽量不依赖外部资源（或统一引用 `assets/` 下的文件）。
- 文件命名建议使用语义化名称，如 `about.html`、`toolbox.html`。
- 如需更新或新增页面，直接在 `pages/` 目录下操作即可。

## 维护说明

- 本仓库不跟踪具体网站内容，仅作为静态页面存档。
- 建议保持目录结构清晰，定期清理不再需要的页面。
- 若使用 GitHub Pages 部署，请确保 `index.html` 或 `pages/` 下的文件可作为静态站点访问。
